3.2.- set functions


The class countertree::cntree_set have the full set of instructions of std::set , plus additional functions, for to resolve the problem exposed  in the point 1.4.2.- Problems associated to the concurrency. Solutions implemented.


The class which encapsulate the code is countertree::set <value_t , cnc,comp_key_t, alloc_t> .

  • value_t  is the type stored in the data structure,
  • cnc is a bool, if true, the data structure implement thread safe mechanism for all the operations, if  false not. The decision depend if you will use the object in a thread or can be accessed by several threads.
  • comp_key_t : is the object for to compare two keys
  • alloc_t is the allocator of the data structure

In order to simplify the use, and make them compatible with the std::set class, we have 4 alias template of this class. 




//***************************************************************************
//                        S E T
//***************************************************************************
template
<   typename value_t,
    typename comp_key_t = std::less<value_t>,
    typename alloc_t = cntree_allocator<void>
>
using set = cntree_set < value_t, false, comp_key_t, alloc_t >;


//***************************************************************************
//                        S E T _ C N C
//***************************************************************************
template
<   typename value_t,
    typename comp_key_t = std::less<value_t>,
    typename alloc_t = cntree_allocator<void>
>
using set_cnc = cntree_set < value_t, true, comp_key_t, alloc_t >;

//***************************************************************************
//                      S E T _ P O O L
//***************************************************************************
template
<   typename value_t,
    typename comp_key_t = std::less<value_t>,
    typename alloc_t = cntree_allocator<void>
>
using set_pool = cntree_set<value_t,true,comp_key_t,suballocator<alloc_t> >;


//***************************************************************************
//                      S E T _ P O O L _ C N C
//***************************************************************************
template
<   typename value_t,
    typename comp_key_t = std::less<value_t>,
    typename alloc_t = cntree_allocator<void>
>
using set_pool_cnc = cntree_set < value_t, true, comp_key_t, suballocator_cnc< alloc_t> >;


You can use different alias for the copy constructor and the assignation, By example :  a set_cnc can received as parameter in the assignation and copy constructor an object set_pool.

If two alias have the same parameter alloc_t, you can use the rvalues in the copy constructor and in the assignation. By example: if the object A is set and the object B is a set_cnc you can use the r values in the assignation  A = std::move(B) ;

The operations with functions, when return true , the action is done. By example : in the function insert_if,  when the function return true, the value is inserted, if false is discarded.
