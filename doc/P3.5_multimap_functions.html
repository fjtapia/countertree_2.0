3.5.- multimap  functions


The class countertree::multimap have the full set of instructions of std::multimap, plus additional functions, for to resolve the problem exposed in the point 1.3. concurrency.


The class which encapsulate the code is countertree::cntree_multimapvector <key_t, data_t , cnc, comp_key_t, alloc_t> . 

The elements stored in the multimap are std::pair <const key_t, data_t>. But only key_t participate in the comparison

  • key_t  is the type which use to compare in the data structure,
  • data_t : contains information but don't participate in the comparison
  • cnc is a bool, if true, the data structure implement thread safe mechanism for all the operations, if  false not. The decision depend if you will use the object in a thread or can be accessed by several threads.
  • comp_key_t : object for to compare two object key_t
  • alloc_t is the allocator of the data structure

In order to simplify the use, and make them compatible with the std::multimap class, we have 4 alias template of this class.

The operations with functions, when return true , the action is done. By example : in the function insert_if,  when the function return true, the value is inserted, if false is discarded.



If two alias have the same parameter alloc_t, you can use the rvalues in the copy constructor and in the assignation. By example: if the object A is multimap and the object B is a multimap_cnc you can use the r values in the assignation  A = std::move(B) ;

The operations with functions, when return true , the action is done. By example : in the function emplace__if,  when the function return true, the value is inserted, if false is discarded.


You have too, new functions of modification for to do thread safe modifications with iterators or with positions in the multimap. There are two new functions insert_or modify. The idea is

  • if the element exist, you do a modification. With the value D in the first, or with a a function M in the second.
  • If don't exist , insert with the value D



std::pair<iterator, bool >   insert_or_modify ( const key_type& x ,const data_t &D )
std::pair<iterator, bool >   insert_or_modify ( const key_type& x ,const data_t &D ,
                                                std::function<void( value_type &)>  M)


The value returned is an iterator to the existing element or the inserted element , and a bool which indicate if true, the element had been inserted , and if false, it had been modified all the elements with that key
