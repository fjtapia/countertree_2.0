2.2.- Vector_tree functions


The class vector_tree have the full set of instructions of std::vector and std::deque, plus additional functions, for to resolve the problem exposed in the point 1.4.2.- Problems associated to the concurrency. Solutions implemented.


The class which encapsulate the code is countertree::cntree_vector <value_t , cnc, alloc_t> .

  • value_t  is the type stored in the data structure,
  • cnc is a bool, if true, the data structure implement thread safe mechanism for all the operations, if  false not. The decision depend if you will use the object in a thread or can be accessed by several threads.
  • alloc_t is the allocator of the data structure

In order to simplify the use, and make them compatible with the vector , or the deque class, we have 4 alias template of this class. 



//***************************************************************************
//             V E C T O R _ T R E E
//***************************************************************************
template
<   typename value_t,
    typename alloc_t= cntree_allocator<void >
>
using vector_tree = cntree_vector < value_t, false, alloc_t >;


//***************************************************************************
//             V E C T O R _ T R E E _ C N C
//***************************************************************************
template
<   typename value_t,
    typename alloc_t= cntree_allocator<void >
>
using vector_tree_cnc = cntree_vector <value_t,true,alloc_t >;

//***************************************************************************
//             V E C T O R _ T R E E _ P O O L
//***************************************************************************
template
<   typename value_t,
    typename alloc_t= cntree_allocator<void>
>
using vector_tree_pool = cntree_vector<value_t,false,suballocator<alloc_t> >;


//***************************************************************************
//             V E C T O R _ T R E E _ P O O L _ C N C
//***************************************************************************
template
<   typename value_t,
    typename alloc_t= cntree_allocator<void>
>
using vector_tree_pool_cnc = cntree_vector<value_t,true,suballocator_cnc<alloc_t> >;

   

You can use different alias for the copy constructor and the assignation, By example :  a vector_tree_cnc can received as parameter in the assignation and copy constructor an object vector_tree_pool.

If two alias have the same parameter alloc_t, tou can use the rvalues in the copy constructor and in  the assignation. By example: if the object A is vector_tree and the object B is a vector_tree_cnc you can use the r values in the assignation  A = std::move(B) ;

The operations with functions, when return true , the action is done. By example : in the function emplace_front_if,  when the function return true, the value is inserted, if false is discarded.


You have too, 4 new functions of modification for to do thread safe modifications with iterators or with positions in the vector_tree.
