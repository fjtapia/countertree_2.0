1.3.- New elements of the Library

 

1.3.1.- Access by position and random access iterators



With the information unordered, we have the vector_tree data structure. This data structure have the same interface than a std::vector and std::deque.  You can access to the elements by their position. You can use iterators like pointers, because they are random-access.

The insertion or deletion don't imply any movement of data. It's only insertion or deletion of a node in the tree. The operations of  insertion, deletion and access to elements are O(log(N)) operations.


Like in the STL vector, you have the function at and the operator[ ] for to access to the elements. The two functions use checked access to the elements



const value_type & at ( size_type Pos ) const      
      value_type & at ( size_type Pos )

const value_type & operator[ ] ( size_type Pos ) const      
      value_type & operator[ ] ( size_type Pos )



Additionally, you have two functions, in order to simplify the work with positions.The functions insert_pos and erase_pos, permit  to you manage the elements by their position, without iterators.


iterator insert_pos ( const value_type & D , size_type Pos);

void erase_pos ( size_type Pos);
void erase_pos ( size_type Prim, size_type Ult);
 


The ordered containers ( set, multiset, map and multimap have, too access by position., but they don't use the operator [] due this operator is used in the map class, and can be confused
Instead of this, we have the function pos, which return the element stored in the position passed as parameter, and if you want an iterator to a position , you have the function iterator_pos

 

//***************************************************************************
//      P O S , I T E R A T O R _ P O S 
//***************************************************************************
      value_type & pos ( signed_type Pos1);
const value_type & pos ( signed_type Pos1 )const;

      iterator iterator_pos ( signed_type Pos1) NOEXCEPT;
const_iterator iterator_pos ( signed_type Pos1) const NOEXCEPT;



1.3.2.- Deletion in the first/last position.

The 5  base classes of countertree ( cntree_vector_tree, cntree_set, cntree_multiset, cntree_map and cntree_multimap ) have operations for to extract elements in the first and last positions. The functions front and back exist only  in std::vector and std::deque, All the others functions are new for all the classes.

These functions permit to implement a concurrent stack or a concurrent queue with the  cntree_vector_tree, cntree_set, cntree_multiset, cntree_map and cntree_multimap in a trivial and safe mode.

All the classes have too, the access by position like the vectors, with the function pos, in the const and non const version, and the functions with iterators iterator_pos and const_iterator_pos, which returns the iterators of the elements in a position in the tree.

The functions with the suffix _if, delete the element if the function return true. The function don't modify the element, only receives then element as parameter and return true or false.


1.3.3.- Iterators


The countertrees have a rich set of iterators ( iterator, const_iterator, reverse_iterator and const_reverse_iterator ). All are random access, for to reach any position don't need travel by any other  position in the same way than the STL iterators.  These iterators are fully compatible with the functions and algorithms of the STL.


If you add 10 to an iterator, the iterator point 10 positions forward, And if you subtract 10 , or add -10, the iterator point 10 positions backward.  You can subtract two iterators and know the distance between them.





You have 4 special places ( rend, begin ,rbegin,end). All kind of iterators can reach the four special places, by example: if you want  with an iterator, reach the rend position have the function it_rend ( ). The preffix of the function indicates the kind of iterator returned.

it_     : iterator
cit_   : const_iterator
rit_   : reverse_iterator
crit_ : const reverse_iterator

You can subtract two iterators for to know the number of elements between them. This arithmetic is valid with ALL the iterators, even with the four special places .

All kind of iterators have a function pos( ) which provide the position in the vector_tree even in the special places.
You can obtain an iterator to the element in a position with the function iterator_pos or const_iterator_pos
