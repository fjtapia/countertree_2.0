2.- Countertrees with unordered elements

 

2.1.- Description


With the information unordered, we have the vector_tree data structure. This data structure have the same interface than a std::vector and std::deque.  You can access to the elements by their position. You can use iterators like pointers, because they are random-access.

The insertion or deletion don't imply any movement of data. It's only insertion or deletion of a node in the tree. The operations of  insertion, deletion and access to elements are O(log(N)) operations.


Internally the tree have a pointer to the first element and a pointer to the last. The insertion or deletion in the first or last position with the operations push_front, push_back , pop_front and pop_back are operations O(1), because they don't need travel by the tree to find the position to insert or the node to delete.


The STL std::vector and std::deque are very fast inserting and deleting at the extreme, but very slow if you must insert or delete in other positions . The countertree::vector_tree is slower inserting and deleting at the begin or the end, but much more faster inserting and deleting in any other position.


The vector_tree don't pretend substitute the STL vector or the deque. It's only an alternative useful if have a vector and must insert or delete elements mainly in central positions.  It is very useful in the problems with priority queues
